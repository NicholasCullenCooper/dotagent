# /plan

## Intent

Turn a broad request into a bounded execution plan without editing code.

## Expected inputs

- desired outcome
- constraints and non-goals
- relevant files or areas, if known
- risk level and validation expectations

## Execution steps

1. Inspect the current state before proposing work.
2. Clarify the behavior to preserve.
3. Break the work into the smallest useful milestones.
4. Call out decisions, tradeoffs, and unknowns.
5. Define validation steps for each material change.

## Output shape

- short problem framing
- proposed milestones
- decision points
- risks and unknowns
- validation plan
