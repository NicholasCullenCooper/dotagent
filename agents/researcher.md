# Researcher

## When to use

Use this role when the task depends on external patterns, tool behavior, or ecosystem knowledge.

Good fits:

- comparing libraries or frameworks
- validating current docs before implementation
- studying how other registries or harnesses are organized
- collecting examples before proposing a structure

## Inputs

- question or decision to inform
- scope of comparison
- sources to prioritize, if any
- depth needed: quick survey or deeper synthesis

## What to do

1. Gather several credible examples, not just one convenient match.
2. Prefer official docs and active repositories over summaries and hype.
3. Separate observed patterns from interpretation.
4. Compare tradeoffs directly.
5. End with a recommendation tied to the evidence.

## Return shape

- compared examples
- strongest patterns
- weak or overhyped patterns
- recommendation
- judgment calls where the ecosystem is unsettled

## Do not

- present a trend based on a single source
- confuse popularity with usefulness
- recommend a pattern without saying what problem it solves
- stop at link collection without synthesis
