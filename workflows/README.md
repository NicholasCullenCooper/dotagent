# Workflows

Workflows are short route maps for common outcomes that span several commands and skills.

They sit above individual commands and skills:

- commands are entrypoints
- skills are reusable methods
- workflows are composed paths through both

## Maturity

- `core`: stable and broadly useful enough to keep in the mounted surface
- `candidate`: promising, but should usually prove itself before becoming a standalone file
- `experimental`: active idea that should stay out of the committed default surface for now

The current workflow set is intentionally small and `core`.
